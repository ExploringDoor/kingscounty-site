⚾ Seed KCSL Firebase

One-time setup tool to push your scraped LeagueLineup data into Firebase Firestore. After seeding, admin.html and the live site will read from Firebase instead of the hardcoded fallback, and edits will persist.

Step 1 — Create the Firebase project

  1. Go to console.firebase.google.com and click Add project.
  2. Name it kcsl-softball (or whatever you want — just copy the generated project ID).
  3. Skip Google Analytics.
  4. In the left sidebar, click Build → Firestore Database → Create database. Pick Start in production mode, then choose a region near Brooklyn (us-east4 or us-east1).
  5. Click Build → Authentication → Get started → Email/Password. Enable Email/Password sign-in so you can add admin users later.
  6. On the project Overview page (gear icon → Project settings → General), scroll to Your apps → Web, click </> to add a web app, call it "KCSL Site", and copy the firebaseConfig object.

Step 2 — Paste the config below

Paste the firebaseConfig object values here. This page will wire up Firebase with those credentials and then push the seed data.

Step 3 — Update Firestore rules

In the Firebase console, go to Firestore Database → Rules and paste this (for now, public-read so the live site works without auth):

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}

Hit Publish. This lets anyone read (for the public site) and only authenticated users write (admin only).
